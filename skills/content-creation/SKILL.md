# 内容创作方法论

> 来源：数字生命卡兹克的内容创作内部分享

## 核心认知

**做内容的本质是讲故事，不是信息搬运。**

信息搬运 = 菜市场买鱼直接扔给别人。内容 = 把鱼做成一道菜端上桌。

如果你只是把别人的东西翻译一遍搬过来，那叫转述，不叫内容。

---

## 三步框架

### 第一步：获取信息（权重 30%）

**关键原则：跨领域获取，像管理投资组合一样管理信息源。**

- 如果你只看自己领域的信息，做出来的东西一定是干巴巴的、没有灵魂的
- 真正的创新几乎不发生在一个领域的内部，总发生在不同领域的交叉点上（美第奇效应）
- T字型人才：上面是广度，下面是垂直深度

**信息源配置建议：**

| 类型 | 作用 |
|------|------|
| 综艺、电影、纪录片 | 最好的叙事教材，学节奏、悬念、情绪设计 |
| 历史、商业 | 提供有厚度的内容弹药库，理解行业的框架 |
| 日常生活观察 | 好内容的底层是对人性的理解，人性在具体的琐碎生活里 |

**操作建议：** 用户提供一个选题时，主动从不相关领域找类比、比喻和参照物。

### 第二步：找角度（权重 60%）

**这是三步里最重要的一步。角度决定一篇内容的生死。**

核心原则八个字：**情理之中，预料之外。**

这就是"陌生化"——让读者对一个他可能已经知道的信息，产生全新的理解。

**操作流程：**

1. 拿到选题后，先想"所有人都会从什么方向切"
2. 他们一定会讲什么
3. **不写那个**
4. 找那个所有人都不会想到、但又合情合理的角度

**经典案例：**
- 七夕大家都在写甜蜜 → 写《七夕，我在民政局排队离婚》
- 大家都在从上往下俯视讲信息差 → 从下往上看，写被折叠在第一层的人

**核心技巧：抛弃第一直觉。** 你第一时间、第一直觉想到的观点一定不能写，因为所有人都能想到。

### 第三步：创作（权重 10%）

创作权重最小，但决定生死。信息拿到了、角度找到了，讲不好故事就全白费。

**两个重点：**

#### 1. 节奏

好的节奏像呼吸，有松有紧。

- **情绪地图**：设计内容的情绪起伏，有波峰有波谷，不能纯平
- **升番技巧**：从sketch喜剧学来，三个爆点一步步往上翻，一浪比一浪高
- **叙事弧线**：从电影学段落安排，从小说学伏笔和回环
- **英雄之旅**：《千面英雄》的节奏模板，非常适合入手

**升番示例：** 第一番让你笑一下，第二番让你笑得更大声，第三番给你完全没想到的东西，全场爆了。

#### 2. 正向价值观

这是做所有内容的底线，决定生死存亡。

- **不懂的东西千万不要说** — 强行碰不懂的领域，被人一扒就完了
- **不要碰敏感话题** — 表达欲有时就是在给自己挖坑
- **弱者思维** — 保持谦逊，承认自己并没有那么牛逼，反而更真诚可信

## 帮助用户写作时的执行流程

当用户请求写内容时，按以下流程执行：

1. **理解选题** — 明确用户要写什么主题、面向什么受众
2. **找角度（最关键）** — 先列出这个主题的"大众第一直觉角度"，然后抛弃它，提出 2-3 个"情理之中、预料之外"的反差角度供用户选择
3. **跨领域嫁接** — 从历史、电影、喜剧、日常生活等领域找类比和比喻，让内容有厚度
4. **设计节奏** — 规划情绪地图：开头钩子 → 情绪升温 → 波谷（制造张力）→ 升番高潮 → 收尾回环
5. **检查价值观** — 确保内容真诚、不造作、不碰红线

## 角度生成模板

当用户给出一个选题时，用以下结构生成角度：

```
主题：{用户选题}

大众第一直觉角度（不要写）：
- {角度1}
- {角度2}

反差角度（推荐）：
- 角度A：{标题} — {为什么这个角度情理之中、预料之外}
- 角度B：{标题} — {为什么这个角度情理之中、预料之外}
- 角度C：{标题} — {为什么这个角度情理之中、预料之外}
```

## 节奏设计模板

```
情绪地图：

开头（钩子）：{制造悬念/抛出反常识的观点}
升温：{展开背景，建立共鸣}
波谷：{制造张力/冲突/困境}
升番1：{第一个爆点/反转}
升番2：{更大的爆点}
升番3（高潮）：{最意想不到但合情合理的点}
收尾：{回环/升华/留白}
```

## 数据参考（公众号）

衡量内容质量的核心指标：

| 指标 | 合格线 | 说明 |
|------|--------|------|
| 常读用户比例 | ~30% | 真正活跃的粉丝占比 |
| 赞阅比 | 2.5% | 1万阅读250个点赞 |
| 转阅比 | 8% | 1万阅读800个转发，底线5% |
| 完读率（2000字） | 40%+ | 长文（5000字+）30%左右 |
